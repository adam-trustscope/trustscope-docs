# TrustScope CLI

The TrustScope CLI (`trustscope`) discovers AI agents in your codebase, monitors LLM calls locally, and wraps MCP servers for governance.

## Installation

```bash
npm install -g @trustscope/cli
```

## Quick Start

```bash
# Scan your project for AI agents
trustscope scan

# Start a local monitoring proxy
trustscope watch --port 4000

# Wrap MCP servers for governance
trustscope mcp wrap
```

---

## Commands

### `scan` — Discover AI Agents

Scans code for AI agent usage, security risks, and governance gaps.

```bash
trustscope scan [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-d, --dir <path>` | Directory to scan (default: current directory) |
| `-g, --github <org>` | Scan a GitHub organization |
| `-r, --repo <owner/repo>` | Scan a single GitHub repository |
| `-f, --format <format>` | Output format: `terminal` (default), `json`, `sarif` |
| `-o, --output <file>` | Write results to file |
| `-j, --json` | Shorthand for `--format json` |
| `-v, --verbose` | Verbose output |
| `--no-color` | Disable colors (CI mode) |
| `--max-repos <n>` | Max repos to scan (GitHub org mode) |
| `--no-cache` | Disable caching (GitHub mode) |
| `--upload` | Upload results to TrustScope Cloud |
| `--send` | Upload results and open onboarding wizard |
| `--project <id>` | Associate scan with a project |

**What it detects:**
- Ungoverned AI agents (LangChain, CrewAI, AutoGen, LlamaIndex, OpenAI Agents, etc.)
- Security risks (hardcoded API keys, dangerous configurations, code execution)
- Compliance gaps (missing audit trails, no human oversight)

```bash
# Scan local directory
trustscope scan -d ./my-project

# Scan GitHub org
trustscope scan -g my-org --max-repos 50

# Scan and upload to cloud
trustscope scan --upload

# CI/CD output
trustscope scan --format sarif -o results.sarif
```

### `watch` — Local LLM Proxy

Starts a local HTTP proxy that intercepts LLM API calls and monitors for loops, velocity spikes, and cost overruns.

```bash
trustscope watch [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-p, --port <port>` | Proxy port (default: 4000) |
| `-t, --timeout <min>` | Session timeout in minutes (default: 10) |
| `--max-rpm <n>` | Max requests per minute (default: 60) |
| `--max-cost <n>` | Session cost cap in USD (default: 10) |
| `--loop-threshold <n>` | Identical prompts before loop detection (default: 3) |
| `--no-loop-detection` | Disable loop detection |
| `--no-color` | Disable colors |

**What it catches:**
- Infinite loops (same prompt repeated)
- Velocity spikes (too many requests/minute)
- Cost overruns (spending above session cap)

```bash
# Start proxy
trustscope watch --port 4000

# Point your agent at it
export OPENAI_BASE_URL=http://localhost:4000/v1
python my_agent.py
```

### `login` / `logout` / `whoami` — Authentication

```bash
# Authenticate with TrustScope Cloud
trustscope login

# Clear stored credentials
trustscope logout

# Show authenticated user and organization
trustscope whoami
```

### `connect` — Interactive Setup Wizard

Guides you through connecting your project to TrustScope Cloud.

```bash
trustscope connect
```

Walks through:
1. SDK integration setup
2. Gateway proxy configuration
3. MCP wrapper configuration

### `init` — Initialize Project Config

Creates a `.trustscope.yaml` configuration file and optionally a GitHub Actions workflow.

```bash
trustscope init [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-d, --dir <path>` | Directory to initialize (default: current directory) |
| `--ci` | Also create `.github/workflows/trustscope.yml` |
| `--force` | Overwrite existing config files |

```bash
# Initialize with CI integration
trustscope init --ci
```

---

### `doctor` — Diagnostics

Checks your setup and reports issues.

```bash
trustscope doctor
```

**What it checks:**
- Authentication status
- API connectivity
- Local configuration validity
- Project setup

### `mcp` — MCP Server Governance

Wrap, list, and unwrap MCP servers for governance monitoring.

#### `mcp wrap`

Patches your MCP client configuration to route servers through TrustScope.

```bash
trustscope mcp wrap [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-c, --config <path>` | MCP config file path |
| `-p, --project <id>` | Project ID for association |
| `-v, --verbose` | Verbose output |

**Auto-detected config locations:**
- **Claude Desktop (macOS):** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Claude Desktop (Linux):** `~/.config/claude/claude_desktop_config.json`
- **Cursor:** `~/.cursor/mcp.json`
- **VS Code:** `~/.vscode/mcp.json`
- **Project-level:** `.mcp.json`

```bash
# Auto-detect and wrap all MCP servers
trustscope mcp wrap

# Wrap specific config
trustscope mcp wrap -c ~/.cursor/mcp.json
```

#### `mcp list`

Show currently wrapped MCP servers.

```bash
trustscope mcp list [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-v, --verbose` | Show server details |

#### `mcp unwrap`

Remove TrustScope wrapper from MCP servers.

```bash
trustscope mcp unwrap [OPTIONS]
```

| Option | Description |
|--------|-------------|
| `-a, --all` | Unwrap all servers |
| `-c, --config <path>` | Config file to modify |
| `-v, --verbose` | Verbose output |

## Environment Variables

| Variable | Description |
|----------|-------------|
| `TRUSTSCOPE_API_KEY` | API key for authenticated operations |
| `TRUSTSCOPE_API_URL` | Override API endpoint (default: `https://api.trustscope.ai`) |
| `GITHUB_TOKEN` | GitHub authentication for org/repo scanning |
| `TRUSTSCOPE_GITHUB_TOKEN` | GitHub token (takes precedence over `GITHUB_TOKEN`) |

---

## Configuration File

The CLI reads from `.trustscope.yaml` in your project root:

```yaml
# .trustscope.yaml
project_id: "proj_abc123"
agent_id: "my-agent"
```

## CI/CD Integration

### GitHub Actions

```yaml
# .github/workflows/trustscope.yml
name: TrustScope Scan
on: [push, pull_request]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @trustscope/cli
      - run: trustscope scan --format sarif -o results.sarif
        env:
          TRUSTSCOPE_API_KEY: ${{ secrets.TRUSTSCOPE_API_KEY }}
      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results.sarif
```

Generate this automatically with `trustscope init --ci`.
