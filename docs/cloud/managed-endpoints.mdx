# Managed Endpoints

Managed Endpoints enable centralized deployment and policy management for Safe Mode across your team.

## Overview

Standard Safe Mode installations are self-managed. Managed Endpoints add:

- **Centralized policies** — Push policies from dashboard
- **Forced compliance** — Local overrides blocked
- **Remote monitoring** — Health status and alerts
- **Automatic updates** — Safe Mode version management

## Pricing

Managed Endpoints are available on the Enforce tier and above:

| Tier | Cost per Endpoint |
|------|-------------------|
| Enforce | $10/endpoint/month |
| Govern | Included (contact sales) |

## Setting Up a Managed Endpoint

### For Team Members

```bash
# Connect as managed endpoint
safemode connect --managed
```

This:
1. Authenticates with TrustScope Cloud
2. Registers device as managed endpoint and issues a **device token** in the format `smdev_XXXX` (e.g., `smdev_a1b2c3d4e5f6`)
3. Downloads and applies team policy
4. Disables local policy overrides

The device token is stored locally and used for all subsequent communication with TrustScope Cloud. It uniquely identifies this endpoint.

### For Administrators

1. Navigate to **Endpoints > Pending**
2. Review pending endpoint requests
3. Approve and assign policy
4. Endpoint receives policy immediately

## Endpoint Status

View endpoint status at **Endpoints**:

| Status | Description |
|--------|-------------|
| **Active** | Connected, policy synced |
| **Offline** | Not connected in >24 hours |
| **Pending** | Awaiting admin approval |
| **Suspended** | Manually suspended by admin |

### Heartbeat Status

Endpoint health is determined by the time since the last heartbeat:

| Status | Condition | Meaning |
|--------|-----------|---------|
| **Online** | < 2 minutes since last heartbeat | Endpoint is healthy and responsive |
| **Degraded** | 2 - 10 minutes since last heartbeat | Endpoint may be experiencing issues |
| **Offline** | > 10 minutes since last heartbeat | Endpoint is unreachable or stopped |

The bridge client sends heartbeats every 60 seconds. If an endpoint transitions to **degraded** or **offline**, an alert is triggered (if configured).

## Policy Enforcement

Managed endpoints enforce policies strictly:

```yaml
# Team policy
enforce_mode: strict

# On managed endpoint:
# - .safemode.yaml is overwritten on sync
# - safemode allow commands are blocked
# - Preset changes require admin approval
```

### Enforcement Levels

| Level | Local Changes | Override Commands |
|-------|--------------|-------------------|
| `strict` | Blocked | Blocked |
| `monitored` | Logged | Logged |
| `flexible` | Allowed | Allowed |

## Remote Management

### Push Policy Update

1. Update policy in dashboard
2. Select **Push to Endpoints**
3. Choose immediate or scheduled push
4. Monitor rollout status

### Remote Commands

Execute commands on managed endpoints:

```bash
# Via API
curl -X POST https://api.trustscope.ai/v1/endpoints/{id}/command \
  -H "Authorization: Bearer $TRUSTSCOPE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "command": "scan",
    "args": ["--fix"]
  }'
```

Available remote commands:
- `scan` — Run security scan
- `status` — Get current status
- `refresh` — Force policy sync
- `restart` — Restart Safe Mode proxy

### Suspend/Revoke

Immediately suspend a managed endpoint:

1. Navigate to **Endpoints > [Endpoint]**
2. Click **Suspend**
3. Endpoint stops processing until unsuspended

To permanently revoke:
1. Click **Revoke**
2. Endpoint must re-register as new device

## Health Monitoring

Managed endpoints report health metrics:

- **Connection status** — Last ping time
- **Policy version** — Currently applied policy
- **Engine status** — Local and cloud engine health
- **Error rate** — Recent detection errors

Configure health alerts:

1. Navigate to **Alerts > Create Alert**
2. Select "Endpoint Health"
3. Choose conditions (offline >1h, high error rate, etc.)
4. Set notification channel

## Bulk Operations

Perform operations on multiple endpoints:

1. Navigate to **Endpoints**
2. Select endpoints (checkbox)
3. Choose action:
   - Assign policy
   - Push update
   - Suspend
   - Export report

## Config Push and Tamper Detection

When a policy is updated, TrustScope pushes the new configuration to all assigned managed endpoints. Each config push includes an `expected_config_hash` — a SHA-256 hash of the policy as it should appear on the endpoint.

The bridge client verifies the hash after applying the config:

```json
{
  "endpoint": "smdev_a1b2c3d4e5f6",
  "config_version": 14,
  "expected_config_hash": "sha256:e3b0c44298fc1c14...",
  "applied_config_hash": "sha256:e3b0c44298fc1c14...",
  "status": "verified"
}
```

If the applied config hash does not match the expected hash, a **tamper detection** alert is raised. This can indicate:

- Local modification of the policy file
- Incomplete config sync
- Filesystem corruption

Tamper events are logged and trigger admin notifications automatically.

## Bridge Client

The **bridge client** is the local agent that runs on each managed endpoint. It handles communication with TrustScope Cloud on three intervals:

| Function | Interval | Description |
|----------|----------|-------------|
| **Event sync** | Every 30 seconds | Uploads buffered trace events and detection results |
| **Policy pull** | Every 5 minutes | Checks for and applies policy updates |
| **Heartbeat** | Every 60 seconds | Reports endpoint health and status |

```bash
# View bridge client status
safemode bridge status

# Output
Bridge Client: running
  Device Token: smdev_a1b2c3d4e5f6
  Last Event Sync: 12s ago (OK)
  Last Policy Pull: 2m 45s ago (OK)
  Last Heartbeat: 8s ago (OK)
  Config Hash: sha256:e3b0c44298fc1c14...
  Events Buffered: 3
```

If the bridge client loses connectivity, events are buffered locally and synced when the connection is restored. The local buffer retains up to 10,000 events before oldest events are dropped.
