# NIST AI Risk Management Framework

The NIST AI RMF provides guidelines for managing AI risks. TrustScope maps directly to its core functions.

## Framework Overview

The NIST AI RMF consists of four core functions:

```
┌─────────────────────────────────────────────────────────────┐
│                     NIST AI RMF                              │
├───────────────┬───────────────┬───────────────┬─────────────┤
│    GOVERN     │      MAP      │    MEASURE    │   MANAGE    │
│  Organization │   Context     │  Analysis     │  Response   │
│  policies     │   mapping     │  assessment   │  actions    │
└───────────────┴───────────────┴───────────────┴─────────────┘
```

---

## GOVERN Function

Establish organizational AI governance.

### GOVERN 1: Policies and Procedures

**TrustScope Support:**
- Define and enforce AI policies
- Version control for policy changes
- Audit trail of policy violations

```python
# Define organization-wide policies
policies = [
    "no-pii",
    "cost-limit-100",
    "content-filter",
    "business-hours-only"
]
```

### GOVERN 2: Accountability Structures

**TrustScope Support:**
- Role-based access control
- Action attribution
- Approval workflows

### GOVERN 3: Workforce Diversity

**TrustScope Support:**
- Bias detection in outputs
- Demographic fairness analysis
- Disparate impact monitoring

**TrustScope Support:**

## Trustworthy AI Characteristics

NIST defines characteristics of trustworthy AI that TrustScope helps achieve:

| Characteristic | TrustScope Feature |
|----------------|-------------------|
| **Valid & Reliable** | Continuous monitoring, drift detection |
| **Safe** | Policy enforcement, blocking |
| **Secure & Resilient** | Threat detection, audit trails |
| **Accountable** | Immutable logging, attribution |
| **Transparent** | Full observability, explainability |
| **Explainable** | Decision logging, factor tracking |
| **Privacy-Enhanced** | PII detection, data minimization |
| **Fair** | Bias detection, fairness metrics |
