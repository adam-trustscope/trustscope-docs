# Supported LLM Providers

TrustScope works with **all major LLM providers** through our Gateway proxy. Your existing code works unchanged - just point to our Gateway URL.

## Provider Overview

| Provider | Gateway URL | Native Format | Status |
|----------|-------------|---------------|--------|
| OpenAI | `gateway.trustscope.ai/v1` | OpenAI | GA |
| [Anthropic](/docs/gateway/anthropic) | `gateway.trustscope.ai/anthropic` | Messages API | GA |
| [Google Gemini](/docs/gateway/google-gemini) | `gateway.trustscope.ai/google` | OpenAI-compat | GA |
| Mistral | `gateway.trustscope.ai/mistral` | OpenAI-compat | GA |
| Cohere | `gateway.trustscope.ai/cohere` | OpenAI-compat | GA |
| Azure OpenAI | `gateway.trustscope.ai/azure` | Azure OpenAI | GA |
| AWS Bedrock | `gateway.trustscope.ai/bedrock` | OpenAI-compat | GA |
| Groq | `gateway.trustscope.ai/groq` | OpenAI-compat | GA |
| Together AI | `gateway.trustscope.ai/together` | OpenAI-compat | GA |
| Perplexity | `gateway.trustscope.ai/perplexity` | OpenAI-compat | GA |
| [Any OpenAI-compatible](/docs/gateway/custom-providers) | `gateway.trustscope.ai/proxy` | OpenAI-compat | GA |

:::tip Beyond the 10 named providers
Any provider that uses the OpenAI chat completions format — xAI (Grok), Fireworks AI, self-hosted vLLM/Ollama, or anything else — works through the **generic proxy** endpoint. See [OpenAI-Compatible Providers](/docs/gateway/custom-providers) for setup.
:::

## Integration Patterns

### Pattern 1: Direct SDK (OpenAI, Anthropic)

Best for: Native SDK users who want minimal changes

```python
# OpenAI - just change base_url
from openai import OpenAI
client = OpenAI(
    base_url="https://gateway.trustscope.ai/v1",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)

# Anthropic - same pattern
from anthropic import Anthropic
client = Anthropic(
    base_url="https://gateway.trustscope.ai/anthropic",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)
```

### Pattern 2: OpenAI-Compatible (Gemini, Mistral, etc.)

Best for: Non-OpenAI providers, unified SDK experience

```python
# Use OpenAI SDK for any provider
from openai import OpenAI

# Gemini
client = OpenAI(
    base_url="https://gateway.trustscope.ai/google",
    api_key="your-google-key",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)

# Mistral
client = OpenAI(
    base_url="https://gateway.trustscope.ai/mistral",
    api_key="your-mistral-key",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)
```

### Pattern 3: Framework Integration (LangChain, CrewAI)

Best for: Framework users who want automatic monitoring

```python
from langchain_openai import ChatOpenAI
from trustscope.langchain import TrustScopeCallback

llm = ChatOpenAI(
    model="gpt-4o",
    base_url="https://gateway.trustscope.ai/v1",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)

# Or use callbacks for more detailed tracking
llm.invoke("Hello", config={"callbacks": [TrustScopeCallback()]})
```

## What Gets Captured (All Providers)

Regardless of provider, TrustScope captures:

| Data | Description |
|------|-------------|
| **Request** | Full prompt, system message, tools |
| **Response** | Generated content, tool calls, stop reason |
| **Tokens** | Input tokens, output tokens, total |
| **Cost** | Calculated from provider pricing |
| **Latency** | Time to first token, total time |
| **Metadata** | Model, provider, agent ID, session ID |
| **Detections** | PII, prompt injection, policy violations |

## Any OpenAI-Compatible Provider

Don't see your provider above? If it uses the OpenAI chat completions format (`messages[]` in, `choices[].message.content` out), it already works via the **generic proxy**:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://gateway.trustscope.ai/proxy",
    api_key="your-provider-key",
    default_headers={
        "X-TrustScope-Key": "ts_live_xxx",
        "X-Upstream-Base-URL": "https://api.your-provider.com/v1",  # ← Provider's real URL
    }
)
```

This works for xAI (Grok), Fireworks AI, Anyscale, self-hosted vLLM/Ollama servers, and any other OpenAI-compatible endpoint. Full guide: [OpenAI-Compatible Providers](/docs/gateway/custom-providers).
