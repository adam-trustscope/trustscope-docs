# Knobs Reference

Every Safe Mode knob controls a single action type and accepts one of three levels:

| Level | Behavior |
|-------|----------|
| `allow` | Action proceeds without interruption |
| `approve` | Action pauses until human approval (terminal, Telegram, or Discord) |
| `block` | Action is denied completely |

Presets set sensible defaults. Override any knob individually in `.safemode.yaml` or at runtime with `safemode allow <knob>`.

:::warning Hardcoded knobs
`terminal.destructive_commands` and `terminal.pipe_to_shell` are always `block` and cannot be overridden. These protect against `rm -rf /`, fork bombs, and piped shell execution.
:::

---

## Terminal (10 knobs)

Controls for shell and subprocess execution.

| Knob | Description | Default |
|------|-------------|---------|
| `command_exec` | General command execution | allow |
| `destructive_commands` | `rm -rf`, `mkfs`, fork bombs | **block** (hardcoded) |
| `network_commands` | `curl`, `wget`, `ssh`, etc. | allow |
| `package_installs` | `npm install`, `pip install`, etc. | approve |
| `sudo` | Elevated privilege commands | block |
| `pipe_to_shell` | Piping output to `sh`, `bash`, `eval` | **block** (hardcoded) |
| `kill_processes` | `kill`, `pkill`, process termination | approve |
| `daemons` | Starting background services | block |
| `cron_jobs` | Creating cron entries | block |
| `max_subprocesses` | Subprocess count limiting | allow |

## Custom

Define your own knobs in `.safemode.yaml`:

```yaml
custom_knobs:
  my_org_approval:
    description: "Requires manager sign-off"
    default: approve
    match:
      tool_pattern: "org_*"
```

## Default Distribution

Across all 92 built-in knobs:

| Level | Count | Percentage |
|-------|-------|------------|
| `allow` | 27 | 29% |
| `approve` | 42 | 46% |
| `block` | 23 | 25% |

The default posture is **approve-heavy** — most potentially impactful actions pause for human confirmation rather than being silently allowed or hard-blocked.

## Overriding Knobs

### In config file

```yaml
# ~/.safemode/config.yaml
overrides:
  terminal:
    package_installs: allow    # skip approval for installs
  git:
    git_force_push: block      # never allow force push
  financial:
    payment_create: block      # block all payments
```

### At runtime

```bash
# Temporarily allow a specific knob
safemode allow terminal.package_installs

# View current resolved config
safemode config
```

### Per-preset overrides

Each preset configures knobs at the category level via `capabilities`. Individual knob overrides in the `overrides` section take precedence:

```yaml
preset: coding
capabilities:
  terminal: scoped_write       # category-level default

overrides:
  terminal:
    sudo: block                # knob-level override wins
    package_installs: allow    # knob-level override wins
```
