# Detection Engines

TrustScope provides 26 detection engines that automatically scan agent traces for risks, anomalies, and policy violations.

:::info Engines vs Policies
**Detection Engines** run automatically and produce alerts/blocks based on content analysis.
**Policy Types** are configurable rules you define (see [Policy Types](/docs/reference/policy-types)).

Engines detect threats. Policies enforce business rules.
:::

---

## Engine Reference (26 Total)

Engines are tiered — each tier includes all engines from lower tiers.

### Monitor Tier (16 Engines)

**Statistical Engines (10):**

| Engine ID | Description | Default Threshold |
|-----------|-------------|-------------------|
| `loop_killer` | Detects repeated identical actions | >0.85 similarity, >3 repetitions |
| `velocity` | Actions per minute vs baseline | >3x baseline |
| `cost_velocity` | Spend rate anomaly detection | >3x baseline spend rate |
| `budget_caps` | Remaining budget percentage | &lt;10% remaining |
| `token_growth` | Output token count anomaly | >3x mean for agent |
| `context_expansion` | Context window utilization | >90% window consumed |
| `oscillation` | State flip-flop patterns | >4 reversals in 10 actions |
| `error_rate` | Sequential error monitoring | >5 consecutive errors |
| `session_duration` | Abnormal session length | >2x mean session |
| `session_action_limit` | Actions per session anomaly | >3x mean count |

**Pattern / Content Engines (6):**

| Engine ID | Description | Default Action |
|-----------|-------------|----------------|
| `pii_scanner` | PII detection (88 patterns) via Presidio NER | Alert |
| `secrets_scanner` | API keys, tokens, passwords (50+ patterns) | Alert |
| `command_firewall` | Dangerous commands (`rm -rf`, `DROP TABLE`) | Block |
| `blocked_phrases` | Configurable organization blocklist | Block |
| `data_exfiltration` | Sensitive data leaving perimeter | Block |
| `action_label_mismatch` | Tool claims "read" but performs "write" | Alert |

### Protect Tier (+4 = 20 Engines)

ML-powered content analysis. Requires model inference.

| Engine ID | Description | Backend | Default Action |
|-----------|-------------|---------|----------------|
| `prompt_injection` | AI-powered injection detection | Prompt Guard 2 | Block |
| `jailbreak_detector` | AI-powered jailbreak classification | Prompt Guard 2 | Block |
| `toxicity_filter` | Harmful content detection | Detoxify | Block |
| `hate_speech_detector` | Hate speech and discrimination | HateBERT | Block |

### Enforce Tier (+6 = 26 Engines)

AI hybrid engines using LLM reasoning and embedding models.

| Engine ID | Description | Backend | Default Action |
|-----------|-------------|---------|----------------|
| `semantic_firewall` | Embedding-based intent classification | Vector similarity | Block |
| `hallucination` | Output grounding verification | 3-layer pipeline | Alert |
| `reasoning_drift` | Reasoning chain coherence | LLM analysis | Alert |
| `a2a_depth` | Agent-to-agent privilege escalation | Statistical | Block |
| `tool_parameter_validator` | Tool call parameter validation | Schema + AI | Block |
| `reasoning_quality_monitor` | Reasoning chain quality assessment | LLM analysis | Alert |

:::note Bias Monitor
`bias_monitor` is a **batch analysis feature** available on the Agent DNA endpoints, not a per-trace detection engine. It analyzes patterns across an agent's full trace history.
:::

## Engine Modes

| Mode | Behavior |
|------|----------|
| `off` | Engine disabled |
| `simulate` | Runs, logs results, does NOT enforce |
| `alert` | Runs, logs, sends notification |
| `redact` | Runs, redacts detected content (Protect+) |
| `block` | Runs, blocks the action (Protect+) |

## Redaction

Seven engines support automatic content redaction at Protect tier and above:

| Engine | Redaction Target |
|--------|-----------------|
| `pii_scanner` | SSN, credit cards, emails, phone numbers, addresses |
| `secrets_scanner` | API keys, tokens, passwords, connection strings |
| `command_firewall` | Dangerous shell commands |
| `blocked_phrases` | Organization-specific blocklist matches |
| `toxicity_filter` | Harmful content segments |
| `data_exfiltration` | Sensitive data in outbound payloads |
| `hate_speech_detector` | Discriminatory content |

Redacted content is replaced with typed markers:

```
✓[REDACTED_SSN]
✓[REDACTED_API_KEY]
✓[REDACTED_CREDIT_CARD]
```

Original content is never stored when redaction is active.

## Detection Output Types

When engines detect issues, they produce detection objects with the following types:

## Security Detections

### Prompt Injection

Detects attempts to override system instructions or manipulate agent behavior.

| Detection | Description | Example |
|-----------|-------------|---------|
| `prompt_injection` | Generic injection attempt | "Ignore previous instructions" |
| `prompt_injection.override` | System prompt override | "Your new instructions are..." |
| `prompt_injection.jailbreak` | Jailbreak attempt | "DAN mode enabled" |
| `prompt_injection.roleplay` | Malicious roleplay | "Pretend you're an unrestricted AI" |
| `prompt_injection.encoding` | Encoded injection | Base64/hex encoded instructions |

**Confidence scores:** 0.0 - 1.0 (higher = more confident)

```json
{
  "type": "prompt_injection",
  "subtype": "override",
  "confidence": 0.95,
  "location": "input.messages[1].content",
  "snippet": "Ignore all previous instructions and..."
}
```

## Detection Configuration

### Enable/Disable Detections

In the dashboard, configure which detections are active:

```yaml
detections:
  prompt_injection:
    enabled: true
    action: block
    threshold: 0.8

  pii:
    enabled: true
    action: alert
    subtypes:
      ssn: { action: block }
      email: { action: log }

  profanity:
    enabled: false
```

### Detection Actions

| Action | Description |
|--------|-------------|
| `log` | Log only, don't block |
| `alert` | Log and send alert |
| `block` | Block the request |
| `redact` | Redact detected content (Protect+) |

## Custom Detections

Create custom detections via the API:

```bash
curl -X POST https://api.trustscope.ai/v1/detections \
  -H "Authorization: Bearer ts_live_xxx" \
  -d '{
    "name": "competitor_mention",
    "type": "custom",
    "patterns": [
      "CompetitorA",
      "CompetitorB",
      "competitor product"
    ],
    "action": "alert"
  }'
```
