# Integration Overview

TrustScope offers **five ingestion points** to capture AI agent activity. Choose based on your use case:

## Quick Comparison

| Method | Best For | Setup Time | Code Changes | Coverage |
|--------|----------|------------|--------------|----------|
| [**Gateway**](#gateway) | Existing apps, zero refactoring | 5 min | 1-2 lines | Automatic |
| [**SDK**](#sdk) | New apps, fine-grained control | 15 min | Decorators | Selective |
| [**CLI**](#cli) | Scripts, testing, CI/CD | 2 min | None | Per-command |
| [**Direct API**](#direct-api) | Custom integrations, any language | 10 min | HTTP calls | Manual |
| [**Frameworks**](#framework-integrations) | LangChain, CrewAI, AutoGen | 10 min | Callbacks | Automatic |

## Gateway

**Proxy your LLM calls through TrustScope. Zero code refactoring.**

### How It Works

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ Your    в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ TrustScope        в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ OpenAI   в”Ӯ
в”Ӯ App     в”Ӯ     в”Ӯ Gateway           в”Ӯ     в”Ӯ Anthropicв”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ     в”Ӯ (captures + logs) в”Ӯ     в”Ӯ Gemini   в”Ӯ
                в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ     в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### When to Use
- вң… Existing applications (no refactoring)
- вң… Multiple LLM providers
- вң… Want automatic capture of everything
- вң… Quick proof-of-concept

### Quick Start

```python
# Change 2 lines - that's it
from openai import OpenAI

client = OpenAI(
    base_url="https://gateway.trustscope.ai/v1",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)

# All calls automatically captured
response = client.chat.completions.create(...)
```

### When to Use
- вң… New applications
- вң… Want fine-grained control
- вң… Need to capture non-LLM actions (tool calls, decisions)
- вң… Building custom agent frameworks

### Quick Start (Python)

```python
from trustscope import TrustScope

ts = TrustScope(api_key="ts_live_xxx")

@ts.observe(agent_id="my-agent")
def process_request(user_input: str):
    # Your agent logic
    response = call_llm(user_input)
    return response

# Or wrap specific actions
with ts.session(session_id="user_123") as session:
    session.log_action("tool_call", {"tool": "search", "query": "..."})
    result = execute_tool()
    session.log_result(result)
```

### Quick Start (Node.js)

```typescript
import { TrustScope } from '@trustscope/sdk';

const ts = new TrustScope({ apiKey: 'ts_live_xxx' });

// Wrap functions
const observedAgent = ts.observe(myAgentFunction, { agentId: 'my-agent' });

// Or manual logging
const session = ts.startSession({ sessionId: 'user_123' });
await session.logAction('tool_call', { tool: 'search' });
await session.end();
```

## CLI

**Command-line interface for scripts, testing, and CI/CD.**

### How It Works

```bash
# Wrap any command with TrustScope monitoring
trustscope run -- python my_agent.py

# Or use as a proxy for curl commands
trustscope proxy --port 8080 &
curl http://localhost:8080/v1/chat/completions ...
```

### When to Use
- вң… Testing and development
- вң… CI/CD pipelines
- вң… One-off scripts
- вң… Quick experiments

### Installation

```bash
# Install via pip
pip install trustscope-cli

# Or npm
npm install -g @trustscope/cli

# Or brew
brew install trustscope/tap/trustscope
```

### Quick Start

```bash
# Set API key
export TRUSTSCOPE_API_KEY=ts_live_xxx

# Run any script with monitoring
trustscope run --agent-id "test-agent" -- python my_script.py

# Watch traces in real-time
trustscope tail

# Check policy before running
trustscope check "What's the weather?" --policy content-filter

# Export recent traces
trustscope export --last 24h --format json > traces.json
```

### CI/CD Integration

```yaml
# GitHub Actions
- name: Run AI Agent Tests
  env:
    TRUSTSCOPE_API_KEY: ${{ secrets.TRUSTSCOPE_API_KEY }}
  run: |
    trustscope run --agent-id "ci-agent" --session-id "${{ github.run_id }}" \
      -- pytest tests/agent_tests.py
```

### When to Use
- вң… Languages without SDK (Go, Rust, Java, etc.)
- вң… Serverless functions
- вң… Custom frameworks
- вң… Maximum flexibility

### Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/v1/observe` | POST | Log an action or LLM call |
| `/v1/enforce` | POST | Check policy before action |
| `/v1/sessions` | POST | Start a session |
| `/v1/sessions/{id}/end` | POST | End a session |
| `/v1/traces` | GET | List traces |
| `/v1/traces/{id}` | GET | Get trace details |

import java.net.http.*;
import java.net.URI;

public class TrustScopeClient {
    private static final String API_URL = "https://api.trustscope.ai/v1/observe";
    private final String apiKey;
    
    public void logObservation(String agentId, String input, String output) throws Exception {

### LangChain Example

```python
from langchain_openai import ChatOpenAI
from langchain.callbacks import BaseCallbackHandler
from trustscope.langchain import TrustScopeCallbackHandler

# Create callback
callback = TrustScopeCallbackHandler(
    api_key="ts_live_xxx",
    agent_id="langchain-agent"
)

# Add to LLM
llm = ChatOpenAI(model="gpt-5.2", callbacks=[callback])

# Or add to chain
chain = prompt | llm | parser
chain.invoke({"input": "Hello"}, config={"callbacks": [callback]})
```

## MCP Server {#mcp-server}

**Model Context Protocol server with 11 governance tools and 18 local detection engines.**

### When to Use
- Claude Desktop, Claude Code, Cursor, VS Code, or any MCP-compatible client
- Local governance without cloud dependency
- Human-in-the-loop approval workflows

### Setup

```bash
# Via CLI (auto-detects and patches MCP client configs)
trustscope mcp wrap

# Or manual вҖ” add to Claude Desktop config
{
  "mcpServers": {
    "trustscope": {
      "command": "npx",
      "args": ["trustscope", "mcp"],
      "env": {
        "TRUSTSCOPE_API_KEY": "ts_live_xxx"
      }
    }
  }
}
```

**Two modes:**
- **Local** (no API key): 18 detection engines, local SQLite evidence store, no cloud sync
- **Connected** (with API key): Local + cloud sync, cloud policies, managed endpoint registration

### 11 MCP Tools

#### Policy & Detection

| Tool | Description |
|------|-------------|
| `trustscope_check_policy` | Check content/action against configured policies. Input: `agent_id`, `action_type`, optional `tool_name`, `tool_args`, `content`. Returns: allow/alert/block decision with detected issues. |
| `trustscope_check_detection` | Run 18 local detection engines on content. Input: `content`, optional `context` (agent_id, session_id, direction), optional `engines[]` to select specific engines. Returns: detection results from all engines. |

#### Evidence & Logging

| Tool | Description |
|------|-------------|
| `trustscope_log_action` | Log an action to the local evidence store with SHA-256 hash chain integrity. Input: `agent_id`, `action_type`, optional `session_id`, `tool_name`, `request_summary`, `response_summary`, `blocked`, `simulated`, `metadata`. Returns: trace record with hash chain entry. |
| `trustscope_list_traces` | Query the evidence store with filters. Input: optional `agent_id`, `session_id`, `limit` (default 100), `offset`, `blocked_only`. Returns: array of traces with pagination. |

#### Approvals

| Tool | Description |
|------|-------------|
| `trustscope_list_approvals` | List pending, approved, or rejected approval requests. Input: optional `agent_id`, `status`, `limit`. |
| `trustscope_approve` | Approve or reject a pending action. Input: `approval_id`, `approved` (boolean), optional `reason`. |

#### Agent Intelligence

| Tool | Description |
|------|-------------|
| `trustscope_get_agent_dna` | Get Agent DNA behavioral fingerprint. Input: `agent_id`. Returns: behavioral profile with z-scores, patterns, risk scores. |
| `trustscope_explain_behavior` | Statistical analysis of agent behavior over a time window. Input: `agent_id`, optional `session_id`, `window_hours` (default 24). Returns: z-scores, tool patterns, risk scoring. |

#### Compliance

| Tool | Description |
|------|-------------|
| `trustscope_list_policies` | List all configured policies. Input: optional `engine` filter. |
| `trustscope_get_compliance` | Get compliance status. Input: optional `agent_id`, `session_id`. Returns: evidence chain integrity + detection summary. |
| `trustscope_get_attestation` | Generate an evidence attestation for external verification. Input: `agent_id`, optional `window_start`, `window_end` (ISO 8601). Returns: unsigned attestation. |

### 18 Local Detection Engines

The MCP server runs 18 detection engines locally (vs 26 cloud engines):

**Statistical (10):** loop_killer, velocity_limit, cost_velocity, budget_caps, token_growth, context_expansion, oscillation, error_rate, session_duration, session_action_limit

**Pattern (8):** pii_scanner, secrets_scanner, command_firewall, blocked_phrases, data_exfiltration, prompt_injection, jailbreak, action_label_mismatch

## Combining Ingestion Points

You can use multiple ingestion points together:

```python
# Gateway for LLM calls
client = OpenAI(
    base_url="https://gateway.trustscope.ai/v1",
    default_headers={"X-TrustScope-Key": "ts_live_xxx"}
)

# SDK for custom actions
ts = TrustScope(api_key="ts_live_xxx")

@ts.observe(agent_id="hybrid-agent")
def my_agent(user_input: str):
    # LLM call goes through Gateway (automatic)
    response = client.chat.completions.create(
        model="gpt-5.2",
        messages=[{"role": "user", "content": user_input}]
    )
    
    # Custom action logged via SDK (manual)
    ts.log_action("business_logic", {"decision": "approved"})
    
    return response
```
