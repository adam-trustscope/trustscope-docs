# Local Development

The fastest useful local setup is the API plus the dashboard. Website, docs, and admin can run alongside them on alternate ports.

## Prerequisites

- Python 3.11
- Node.js 20+
- npm
- Docker Desktop or a compatible Docker runtime
- PostgreSQL and Redis only if you are not using Docker for infra

## Recommended local ports

| Service | Local URL | Notes |
|---|---|---|
| API | `http://localhost:8000` | FastAPI app and OpenAPI docs |
| Dashboard | `http://localhost:3000` | Main customer app |
| Website | `http://localhost:3001` | Run with `next dev --port 3001` |
| Docs | `http://localhost:3002` | Run with `docusaurus start --port 3002` |
| Admin | `http://localhost:3003` | Run with `next dev --port 3003` |

## Boot order

### 1. Start backend dependencies

```bash
cd trustscope-api
cp .env.example .env
docker-compose up -d postgres redis
```

### 2. Start the API

```bash
cd trustscope-api
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload
```

Validate:

```bash
curl http://localhost:8000/health
open http://localhost:8000/docs
```

### 3. Start the dashboard

```bash
cd trustscope-dashboard
cp .env.example .env.local
npm install
npm run dev
```

Minimum local dashboard config:

```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxx
CLERK_SECRET_KEY=sk_test_xxx
```

Useful local checks:

- `/api-check`
- `/dashboard`
- `/onboarding`

### 4. Start optional frontends

Marketing site:

```bash
cd trustscope-website
npm install
npm run dev -- --port 3001
```

Docs site:

```bash
cd trustscope-docs
npm install
npm run start -- --port 3002
```

Admin console:

```bash
cd trustscope-admin
npm install
API_URL=http://localhost:8000 ADMIN_API_KEY=your_admin_key npm run dev -- --port 3003
```

## Local environment notes

- Use the checked-in example files as the starting point. Do not commit generated `.env` or `.env.local` files.
- `trustscope-dashboard` prefers relative API calls in production and only uses a direct base URL for localhost.
- `trustscope-api` expects PostgreSQL and Redis even for basic boot.
- All Next.js apps default to port 3000. Run only one on the default port at a time.

## What to run for common tasks

| Task | Repo | Command |
|---|---|---|
| API tests | `trustscope-api` | `pytest` |
| API targeted security tests | `trustscope-api` | `pytest tests/test_import_workflow_security.py tests/test_csrf_protection.py tests/test_ssrf_protection.py -q` |
| Dashboard tests | `trustscope-dashboard` | `npm run test:run` |
| Docs build | `trustscope-docs` | `npm run build` |
| Website build | `trustscope-website` | `npm run build` |
| Admin build | `trustscope-admin` | `npm run build` |

## Local smoke checklist

- `GET /health` returns healthy or degraded instead of 500.
- FastAPI docs load at `/docs`.
- Dashboard can reach the API and render `/api-check`.
- Auth flow is wired to the intended Clerk environment.
- Website waitlist requests do not point at a dead API URL.
